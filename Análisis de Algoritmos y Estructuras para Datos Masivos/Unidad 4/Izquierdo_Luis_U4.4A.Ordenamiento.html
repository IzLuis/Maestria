Bubble Sort

El algoritmo de ordenamiento conocido como "Bubble Sort" se basa en el ordenamiento de pares de elementos adyacentes en una lista. Revisa los pares y, si estn en el orden incorrecto, les intercambia de lugar, lo cual se repite hasta que la lista est correctamente ordenada.

A pesar de que este algoritmo es el ms sencillo de implementar, es ineficiente para listas de un tamao mayor, teniendo una complejidad temporal de $O(n^2)$.

En la funcin para el bubble sort, se tomar de entrada la lista que se desea ordenar y se guardar en una variable la longitud de la misma.

Se har un recorrido por todos los elementos de un arreglo y se declarar falso una variable que indica si se cambi de lugar el elemento a analizar.

Dentro de este recorrido, por el arreglo de nuevo, esta vez siendo de 0 a la longitud de la lista, menos el valor i, menos 1. Esto debido a que el ltimo elemento del valor i ya est en su lugar. Si el valor encontrado es mayor que el siguiente elemento, se cambia de lugar. En caso de que no se cambien de lugar, se rompe el loop ms interno y se pasa al siguiente elemento de la lista, ya que el valor actual ya est en su lugar.

In[108]:
import copy
def bubbleSort(lista):
  ordenamiento = copy.deepcopy(lista)
  n = len(ordenamiento)
  comparaciones = 0

  # Se recorren todos los elementos de la lista
  for i in range(n):
    swapped = False

    # El ltimo elemento de i ya estn en su lugar
    for j in range(0, n-i-1):

      # Se cambia si el elemento j es ms grande que el siguiente elemento
      comparaciones += 1
      if ordenamiento[j] > ordenamiento[j+1] :
        ordenamiento[j], ordenamiento[j+1] = ordenamiento[j+1], ordenamiento[j]
        swapped = True

    # Si no hubo cambio, se rompe el loop interno
    if swapped == False:
      break
    
  return ordenamiento, comparaciones

Insertion Sort

El algoritmo de insercin compara los elementos con el inmediato anterior e intercambia sus lugares si el anterior es mayor (o se repite hasta que alcance el principio de la lista o encuentre uno mayor).

Este algoritmo es simple de implementar y es eficiente para listas pequeas o aquellas que estn casi ordenadas. Su complejidad temporal es cuadrtica $O(n^2$)

La funcin toma como parmetro la lista a ordenar. Lo primero que esta funcin har ser recorrer la lista desde el segundo elemento hasta el final (longitud de la lista), ya que se considera que el primer elemento ya est ubicado correctamente.

Guarda en la variable keyel valor actual que se insertar y compara el valor de key con los elementos a su izquierda, hasta que est en su posicin correcta y lo inserta en la tabla.

In[110]:
def insertionSort(lista):
  ordenamiento = copy.deepcopy(lista)
  comparaciones = 0
  #Se recorre del segundo elemento de la lista hasta el final.
  for i in range(1, len(ordenamiento)):
    #Se guarda el valor actual a insertar
    key = ordenamiento[i]
    j = i-1
    #Se compara el valor de key con los elementos a su izquierda 
    # hasta que encuentre su posicin correcta y lo inserta.
    while j >= 0 and key < ordenamiento[j]:
      ordenamiento[j+1] = ordenamiento[j]
      j -= 1
      comparaciones += 1
    ordenamiento[j+1] = key
  return ordenamiento, comparaciones

Merge-sort

Como destacan varias bibliografas, el algoritmo merge-sort utiiza el mtodo de "divide y conquista". En cada iteracin ordena un arreglo A[p:r], iniciando por el arreglo entero A[1:n] y volvindolos arreglos cada vez ms pequeos.

Se divide el arreglo A[p:r] para que se ordene en dos arreglos adyacentes, cada uno con la mitad del tamao, quedando los arreglos A[p:q] y A[q+1:r]. De ah, ambos lados se ordenarn haciendo dividiendose de manera recursiva como se vio anteriormente. Finalmente se combinaran los dos arreglos A[p:q] y A[q+1:r] en A[p:r], lo cual ser nuestra respuesta final.

Este algoritmo, a pesar de ser muy eficiente con complejidad temporal $O(nlogn)$, puede no ser muy eficiente por el espacio adicional que requiere al generar las nuevas listas cada que parte la lista principal a la mitad.

En el caso de este algoritmo de ordenamiento, se crearon dos funciones. La primera es como tal el algoritmo merge-sort, que toma de entrada una lista.

Lo primero que se evala es si la lista tiene un elemento o menos, en ese caso ya est ordenada y se devuelve la lista. Posteriormente se encuentra la mitad de la lista y se ivide en izquierda o derecha usando la misma funcin recursivamente. Finalmente se regresa la combinacin de ambos lados por medio de la funcin merge.

La funcin merge compara los elementos de las dos listas y los agrega a una lista de resultado

def mergeSort(lista):
    ordenamiento = lista
    if len(ordenamiento) <= 1:
        return ordenamiento, 0

    medio = len(ordenamiento) // 2
    izquierda, compIzq = mergeSort(ordenamiento[:medio])
    derecha, compDer = mergeSort(ordenamiento[medio:])
    resultado, compRes = merge(izquierda, derecha)
    return resultado, compIzq + compDer + compRes

def merge(izquierda, derecha):
    resultado = []
    i = j = 0
    comparaciones = 0

    while i < len(izquierda) and j < len(derecha):
        comparaciones += 1
        if izquierda[i] <= derecha[j]:
            resultado.append(izquierda[i])
            i += 1
        else:
            resultado.append(derecha[j])
            j += 1

    # Agregar los elementos restantes (si los hay)
    resultado.extend(izquierda[i:])
    resultado.extend(derecha[j:])
    return resultado, comparaciones

Quick Sort

Este algoritmo toma un elemento "pivote" y crea dos arreglos, aquellos menores al elemento pivote y los mayores a este elemento, tambin ordena recursivamente estos arreglos.

Este algoritmo es muy eficiente para listas grandes, teniendo una cumplejidad temporal de $O(n log n)$.

In[114]:
import random
def quickSort(arr):
    def partition(arr, low, high):
        # Elige un pivote aleatorio para mejorar el balance
        pivot = random.randint(low, high)
        arr[pivot], arr[high] = arr[high], arr[pivot]
        i = low - 1
        comparaciones = 0
        for j in range(low, high):
            comparaciones += 1
            if arr[j] <= arr[high]:
                i += 1
                arr[i], arr[j] = arr[j], arr[i]
        arr[i + 1], arr[high] = arr[high], arr[i + 1]
        return i + 1, comparaciones

    def quick_sort_helper(arr, low, high):
        if low < high:
            pi, comparaciones = partition(arr, low, high)
            izqComp = quick_sort_helper(arr, low, pi - 1)
            derComp = quick_sort_helper(arr, pi + 1, high)
            return comparaciones + izqComp + derComp    
        return 0

    comparaciones = quick_sort_helper(arr, 0, len(arr) - 1)
    return arr, comparaciones

import time
def getTime(funcion_ordenamiento, lista):
  itTimes = []
  comparaciones = []
  for arreglo in lista.values():
    inicio = time.process_time()
    ordenada, comp = funcion_ordenamiento(arreglo)
    fin = time.process_time()
    comparaciones.append(comp)
    total = fin - inicio
    itTimes.append(total)
  promedio = sum(itTimes)/len(itTimes)
  promCom = sum(comparaciones)/len(comparaciones)
  return promedio, ordenada, promCom

Conclusiones

En trminos generales, se puede observar que en los casos de los 4 proporciones para las listas perturbadas se observ el siguiente comportamiento:

  • Insertion Sort fue el algoritmo que tomo menos tiempo y comparaciones en todos los casos
  • Quick Sort fue el algoritmo ms lento y que tom ms comparaciones.
  • En todos los casos el orden del ms rpido al ms lento fue:
    1. Insertion Sort
    2. Bubble Sort
    3. Merge Sort
    4. Quick Sort

Es interesante analizar los resultados ya que, en teora, Quick Sort debera ser la funcin de Ordenamiento ms rpida de todas, lo mismo con Merge Sort; el retraso que se encontr en estos dos se puede deber a varias razones como el tamao de los datos y la distribucin de los elementos, sin embargo, se considera que el error fue de implementacin, ya que al ser la primera vez que se implementan estos algoritmos, la implementacin no fue tan limpia como podra hacerse al conocer de mejor manera los algoritmos.

Las dificultades encontradas en estos experimentos tuvieron que ver principalmente con el mbito de programacin, ya que, al desconocer como Python utiliza las listas que se le pasan a cada funcin, constantemente se encontraban errores distintos al procesar los diferentes algoritmos. Uno de estos errores fue que en el Insertion Sort el nmero de comparaciones siempre devolva cero; otro fue que al ejecutar ms de una vez el mismo algoritmo, cambiaban sus resultados y al ejecutarlo de nuevo, siempre se quedaban con aquellos de la segunda ejecucin. La mayor parte de estos errores se solucion haciendo "deep copies" de las listas al pasrselas a la funcin de ordenamiento.
