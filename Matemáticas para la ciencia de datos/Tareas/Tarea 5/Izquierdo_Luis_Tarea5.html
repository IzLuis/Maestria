A veces, esto puede ser un problema muy difcil de resolver, porque no est claro co mo el esfuerzo de la CPU depende de los parmetros. En este ejercicio, consideraremos un programa muy simple para ilustrar cmo se pueden generar estimaciones de tiempo. Supongamos que queremos una solucin numrica del problema

$y'(t) = e^{y(t)}$, $y(0) = 0$, $0 \le t \le 1.$

Si intentamos resolver (1) usando el esquema estndar de Euler, tenemos que

$y_{k+1} = y_k + \Delta te^{yk}, k = 0,1,n1.$

con $y_0 = 0$. Aqu, $\Delta t = 1/n$, donde $n > 0$ es un nmero entero. En la Tabla siguiente hemos enumerado el tiempo de CPU que necesita un sencillo programa en C para calcular $y_n$ en el tiempo $t = 1$ en un procesador Pentium III de 600 MHz.

Inciso 1

Para obtener la funcin lineal $c(n) = \alpha + \beta n$, por medio del mtodo de mnimos cuadrados, es necesario tener en cuenta la manera de calcular ambos.

$\beta = \frac{\sum {xy} - \frac{(\sum x) (\sum y)}{n}}{\sum {x^2} - \frac{(\sum {x})^2}{n}}$

$\alpha = \frac{\sum y}{n} - \beta \frac{\sum x}{n}$

Para facilitar los clculos, se hizo una tabla con las sumatorias de dichos casos

n CPU Time yn XY X^2
100000 0.05 9.9181 0.495905 98.36870761
200000 0.09 10.549 0.94941 111.281401
300000 0.13 10.919 1.41947 119.224561
400000 0.18 11.183 2.01294 125.059489
$\sum{}$ 0.45 42.5691 4.877725 453.9341586

Con esto, se puede calcular $\alpha$ y $\beta$.

$\beta = \frac{134000 - \frac{(1000000)(0.45)}{4}}{300000000000 - \frac{(1000000)^2}{4}} = \frac{21500}{50000000000} = 0.00000043$

$\alpha = \frac{0.45}{4} - 0.00000043\frac{1000000}{4} = 0.005$

La funcin lineal quedara de la siguiente manera:

$c(n) = 0.005 + 0.00000043n$

Evaluando para los valores de $n$ dados originalmente:

n CPU Time Evaluacin
100000 0.05 0.048
200000 0.09 0.091
300000 0.13 0.134
400000 0.18 0.177

Se observa que son bastante similares los evaluados a los originales, por lo cual se concluye que la ecuacin es correcta

Para obtener el tiempo de CPU para cuando n es igual a $10^6$ y $10^7$, simplemente sustituimos el valor de n en la ecuacin obtenida:

$c(n) = 0.005 + 0.00000043(10^6)$

$c(n) = 0.005 + 0.00000043(10^7)$

Siendo los resultados:

n CPU Time
1000000 0.435
10000000 4.305

import matplotlib.pyplot as plt
import numpy as np

n = [100000, 200000, 300000, 400000]
cpu_time = [0.05, 0.09, 0.13, 0.18]

def c(n):
    return 0.005 + 0.00000043 * n

n_ecuacion = np.linspace(min(n), max(n), 100)

plt.plot(n, cpu_time, 'o', label='CPU Time')  # Solo puntos, sin lnea
plt.plot(n_ecuacion, c(n_ecuacion), label='c(n)')
plt.xlabel('n')
plt.ylabel('Tiempo')
plt.title('Ecuacin VS Puntos originales')

plt.legend()
plt.show()

Para el caso cuadrtico, se quiere obtener la ecuacin de la forma $p(t) = \alpha + \beta t + \gamma t^2$. Esta se puede encontrar resolviendo las ecuaciones:

$\alpha n + \beta \sum{t} + \gamma \sum{t^2} = \sum{y}$

$\alpha \sum{t} + \beta \sum{t^2} + \gamma \sum{t^3} = \sum{xy}$

$\alpha \sum{t^2} + \beta \sum{t^3} + \gamma \sum{t^4} = \sum{x^2y}$

A = np.array([[n, sumt, sumt2],
              [sumt, sumt2, sumt3],
              [sumt2, sumt3, sumt4]])

b = np.array([[sumy],
              [sumxy],

# Resolver usando la inversa de A
solution = np.linalg.inv(A) @ b

# Extraer los valores
